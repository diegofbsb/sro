5.3 Arquivos de Configuração

O Arquivo sroserver.conf – contém os itens configuráveis para determinar o comportamento do aplicativo SROserver.jar.

Esse arquivo só precisa ser editado se os valores default precisarem ser alterados, a saber, se o diretório de instalação utilizado não foi o padrão conforme definido anteriormente neste procedimento ou se o diretório do CTS não for padrão (\cts).

syncInterval

Para manter a independência entre o SRO e os demais sistemas que a este estão integrados,
principalmente ao processo NETOPE, foi desenvolvida a sincronização entre pastas. O NETOPE, que gera diversos arquivos, entre eles “uacc” para o cadastro de usuários e controle de acesso nos coletores, dispõe deste arquivo em duas pastas: to send, que recebe arquivos de retorno do coletor através do fromope. O SROserver de tempos em tempos, realiza uma cópia do arquivo “uacc” disposto no folder to send para o folder \SROserver\upload; em segundo momento, copia os arquivos que o coletor dispôs em \SROserver\download, passa por uma formatação no próprio SROserver e entrega-os em \SROserver\final para que a sincronização encarregue-se de copiá-los no folder fromope mesmo folder \SROserver\upload. Isto acontece em tempos regulares, determinado pela variável syncInterval que está em segundos.

3600

5.4 Arquivo AutoIncFile.txt

Este arquivo contém um contador para a função de Baixa de Distribuição Externa e se auto-incrementa a cada evento desta função. Inicialmente recebe o valor ADEL=1.
